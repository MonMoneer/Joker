Mobile Fullscreen Mode

How should the game handle mobile orientation and fullscreen?

A

Auto Fullscreen on Game Start

When a game starts, the app requests fullscreen mode and locks to portrait (vertical). Bottom browser bar hides. User taps "Exit" to leave fullscreen.

How it works: Uses the Fullscreen API + Screen Orientation API.
Portrait lock ensures cards are readable.
A small "exit fullscreen" button stays in the corner.
B

PWA Install Prompt + Fullscreen

Show an "Install App" banner on first visit. Once installed as PWA, it always opens fullscreen in portrait — no browser chrome, feels native.

How it works: PWA manifest with display: standalone and orientation: portrait.
Home screen icon, splash screen, no URL bar.
Works offline after install.
C

Both: PWA + In-Game Fullscreen Toggle

Install as PWA for the best experience. But also add a fullscreen toggle button for users who play in the browser.

Best of both worlds:
• PWA users get native-like experience automatically
• Browser users can tap "⛶" to go fullscreen during gameplay
• Portrait lock on game screens, free rotation on menus

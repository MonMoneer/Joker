Game Table Layout — Which approach?

The current layout uses absolute positioning which breaks on different screen sizes. Here are 3 alternatives.

A: CSS Grid (Recommended)

5 rows × 3 columns. Each player gets their own grid zone — no overlapping ever. Bid buttons sit in row 4 between info and cards. Scales naturally to any screen size.

Pros

  • Zero overlap guaranteed
  • Adapts to any viewport
  • Each zone has guaranteed space

Cons

  • Less "floating table" feel
  • Side players get narrow columns on mobile

B: Flexbox Vertical Stack

Stack everything vertically with a horizontal middle row for left/trick/right. Similar to Grid but uses flexbox nesting.

Pros

  • Simple CSS, easy to understand
  • Natural document flow

Cons

  • Middle row height calculation tricky
  • Less precise control than Grid

C: Keep Absolute (like jok.ge)

Keep the current approach but fix the specific overlap issues. Reduce opponent card sizes, add percentage-based positioning, clamp elements to stay in bounds.

Pros

  • Looks most like a real card table
  • Minimal code changes

Cons

  • Same fundamental problem — hard to prevent overlaps
  • Needs breakpoint-specific positioning
  • Already failed once
